## Speed-up VR development time üöÄ

FastVR is based on [WRMHL](https://github.com/relativty/wrmhl) and built for [Relativ](https://github.com/relativty/Relativ). Using this package you can easily create VR games compatible with [Relativ Headset](https://github.com/relativty/Relativ), an Oculus-like VR headset that cost 100$.

FastVR can be deeply customized without breaking Relativ compatibility. You can use the default distorsion shader, or implement your own. We both recommand default shader and [GVR](https://github.com/googlevr/gvr-unity-sdk). You can also change the WRMHL protocol.

## How to Install
You can either install it using [Git](https://git-scm.com/) or direct [Download](https://github.com/relativty/fastVR-sdk/archive/master.zip). Or from the <strong>command line</strong>:

```bash
# Clone this repository
$ git clone https://github.com/relativty/fastVR-sdk
```

### Import the Package ! üì¶
**Assets**, **Import Package** and **Custom Package..** and you're good to **GO !** ‚ö°Ô∏è

<img src="/img/unity-package.gif">

### Important ! ‚ö† Change the .NET 2.0 Subset to .NET 2.0 ‚ö†

In order to do that : Go to **Edit** ‚û≠  Then **Project Settings** ‚û≠ **Player**, and under **Other Settings** find an option that reads **Api Compatibility Level** and change it from **_".NET 2.0 Subset" to ".NET 2.0"._**
